Create a new texture for Quake 2

  • In Wally, import your source image as an 8-bit indexed bitmap using the Quake 2 palette. This is important. You have to actually import a file that uses the correct color information, otherwise the exported file will be wrong.
  • Go File -> Export... Change the file format to "Quake2 Texture (*.wal)" and click 'Save'.

    • Create an 8-bit indexed (paletted) bitmap

  • Depending on what game you are editing, you may be required to provide textures as 8-bit paletted bitmaps.
  • Use a 32-bit .PNG for your source image - that's what these tutorials are based around:

    • If your source image has > 16 colours:

  • In GIMP, import your source image.
  • Go to Image -> Mode -> Indexed... You can either have GIMP generate a palette or import a custom palette (some games, like Quake 1 & 2, have a fixed 256-color palette that all textures share). Click 'Convert'.
  • Click File -> Export As... Change the file format to "Windows Bitmap (*.bmp)" and click 'Export'.
  • Important: Under 'Compatibility Options', check "Do not write color space information". Click 'Export'.

    • If your source image has <= 16 colours:

  • GIMP won't work, because you cannot force it to export a bitmap with more colour information than is required.
  • Using Wally, import your source image.
  • Optionally, change the color palette to one you require (e.g. Quake 2).
  • Click File -> Export... and select Windows Bitmap (*.bmp) as the file format. Wally will automatically export it as an 8-bit bitmap.

    • Import a color palette to GIMP

  • Because it's not obvious...
  • Go to Windows -> Dockable Dialogues -> Palettes. Right click on the list and select 'Import Palette...'.
